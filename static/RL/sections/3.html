3.2.1. Notes on implementation

  • The rewards should be treated as constant and not differentiable (the environment is not differentiable). Use torch.stack() to accumulate the log probabilities while retaining requires_grad=True.

  • For sampling from the policy, use

    dist = Categorical(logits=logits)
    logp = dist.log_prob(a)

    Reasoning: PyTorch computes log_softmax internally in a stable way. You avoid forming tiny probabilities explicitly. This is the standard trick in classification too.

  • To reduce the variance, subtract the returns to go from their mean. If we don’t do this, then all gradients push probabilities up since all are positive. Instead we want to estimate advantages:

    def train(self, optimizer: torch.optim.Optimizer,
                num_episodes=10, T=100, γ=0.99):
        model = self.model

        model.train()
        for _ in tqdm(range(num_episodes)):
            loss = 0.0
            moves, rewards, logps, _ = self.run_episode(T)
            
            # Returns to go
            returns = []
            for t in range(T):
                G_t = 0.0
                for k in range(T - t):
                    G_t += (γ ** k) * rewards[t + k]
                returns.append(G_t)
            returns = torch.tensor(returns, dtype=torch.float32)
            
            # REINFORCE loss
            loss = - (logps * (returns-returns.mean())).mean()

            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
