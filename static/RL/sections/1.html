1. Introduction

1.1. Markov Decision Processes

We begin with a model of the environment.

Definition 1. An MDP is a 6-tuple where is the set of valid states, is the set of valid actions, is the reward function, is the transition probability function, and is the starting state distribution. We have a discount factor as well.

Given a fully-known MDP, we want to maximize expected long-run reward. When the action space is discrete, we parameterize a categorical policy which outputs logits. For a continuous action space we parameterize a diagonal action policy which outputs a mean and diagonal (log)covariance matrix.

1.3. Bridge from MDP to RL

We assume the environment is an MDP with state space and action space .

  • Unlike dynamic programming, the agent does not have access to the transition kernel or reward function , and does not require an explicit enumeration of ; instead, it observes sampled transitions generated by interacting with the environment under its current policy.
  • The action space is assumed to be known.
  • The agent need not know the full set of reachable states.

In partially observed settings, may be latent, and the agent instead conditions on observations (and possibly history ) that is treated as approximately Markov.

The objective is to learn a policy that maximizes expected return.
