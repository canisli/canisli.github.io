1. Introduction

Anything can be a string of bits: data, numbers, instructions, etc.

A high-level programming language is a language with strong abstraction from the specifics of processor architecture. It compiles down to assembly code, which compiles to machine code (bits).

2. Number Representation

  • A byte can be represented as 2 hexadecimals. For example, 0xFD = 0b11111011.

  • Examples of bit representations:

    • ASCII: 8-bit representation of uppercase, lowercase, and punctuation
    • Unicode: 8-bit/16-bit/32-bit representation of all symbols including emojis.
    • HTML color codes: 24-bit representations representing RGB.
    • IPv4 and IPv6 are 32-bit and 64-bit representations of device addresses on the internet.

  • Strategies to encode signed integers:

    1. We can try to use a bit to represent integer sign, but the overflow pattern is weird2.
    1. One’s complement: A better approach is to represent a negative number by complementing the bits of its positive representation. The leftmost bit is still effectively the sign bit.

    1. Two’s complement: One’s complement has two ambiguous zeros: 0000 and 1111. Shift all the negative mappings left by one so that 1111 represents −1 and 1000 represents −8.

3. C

3.1. Introduction

  • C is a functional language that gives you low-level control to do manual memory management and write concurrent/parallel programs.

    • For example, in Python you can’t write a function that adds one to an integeer.

  • Recall from Figure 1 that C is compiled into machine code. An executable is composed of this binary machine code. An intrepreter (runs as machine code) instead reads program’s instructions and performs the corresponding actions step by step without ever translating your whole program into machine code.

    • Java compiles into Java bytecode and the JVM does a combination of interpreting while translating some stuff to machine code to run faster.
    • Python compiles into Python bytecode. Python interpreter runs them step by step.
  • Rationale: Compiled languages can be transferred more easily between architectures. You just need to rewrite the compiler, e.g. Apple in 2020 from Intel to ARM.
  • Static typing helps the compiler allocate bits in advance for variables. For example, sizeof(arg) is resolved by the compiler rather than treated as a function call.

  • Variables

    • The C standard does not define the absolute size of all integer types. It depends on the machine.
    • C variable declarations do not initialize variables to default values.

  • Syntax

    • Preprocessor Macros are string replacement
    • Header files allow you to share functions and macros across different source files. During compilation the preprocessor literally pastes the contents into that file.

3.2. Memory and Pointers

  • We treat memory as a long grid of bytes, each with an address and a value:

    • On a 64-bit computer, each address is 8 bytes. This corresponds to unique addresses and a ceiling of bytes in memory.
    • A 32 computer has a ceiling of 4 GiB in memory.
    • [TODO] I think we will go over later how variables get allocated and the interaction of the compiler and the machine.
  • The address of a variable is the address of the first byte of its memory block.

  • Pointers are variables that contain the address of another variable . In C they are typed i.e. p points to a 32-bit unsigned integer so we go there and read 4 bytes. void *p defines a generic pointer.

    • Dereferencing refers to following a pointer to its object and can be used for reading and modification.

    Example 1.

    int *p;
    int x = 3;
    p = &x; // If we remove this line then the pointer points to a random part of memory.
    printf("p points to %d\n", *p);
    *p = 5;

    *p denotes both declaring a pointer variable and dereferencing it. & is the address operator. Note that when we set p to point towards x, we don’t use an asterisk.

    • C is pass-by-value, so in order to modify a variable, we have to do something like:

      void add_one(int *p) {
        *p = *p + 1;
      }
      int main() {
        int y = 3;
        add_one(&y);
      }
  • The pointer to the all-zero address is the NULL pointer, which is reserved (not permitted to read or write to that address).
  • ptr + n adds n*sizeof(*ptr) bytes to the memory address stored in ptr.
  • If you wish to add one to a pointer in a function then you need to use a double pointer. See array section for an example.

3.3. Arrays

  1. Array declarations set aside contiguous blocks in memory.

  2. In C, an array expression decays to a pointer to its first element. So the following code works as expected:

    #include <stdio.h>

    void increment_ptr(uint32_t **h) {
      /*
      uint32_t ** declares h as a pointer to a pointer, 
      so we modify the pointer via *h. 
      */
      *h = *h + 1;
    }

    int main() {
      uint32_t arr[3] = {50, 60, 70};
      uint32_t *q = arr; // no need for &arr
      increment_ptr(&q);
      printf("*q is %d\n", *q);
      return 0;
    }
  3. Yet arrays are not pointers because you cannot do things like add to them like you would with a pointer. They are a derived type formed from an element type and a size.

Caveats:

  • When used as parameters for functions, int arr[] becomes syntactic sugar for int *arr (pointer), so you should specify size as another parameter for the function to know3
  • C does not have array-bounds exceptions. There is no runtime check. Accessing outside an array’s bounds is undefined behavior. Usually C computes the address as if the array were infinite.

3.4. Strings

  • A C string is just an array of characters followed by a nul terminator (byte of all 0s.), e.g. char my_str[] = {'e', 'x', 'a', 'm', 'p', 'l', 'e', '\0'};. You can also create the string using double quotes, in which the null terminator is implicitly added: char my_str[] = "example";
char *my_immutable_str = "Hello"; // immutable, points at read-only memory
char my_str[] = "hello";                   // mutable, created on the stack
