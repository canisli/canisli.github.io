1.3. Pytorch

  • Use Einsum: assign letter variables to each axis of the input tensors, and then explicitly write down which axes end up in the output tensor.

    # Ordinary matrix multiplication written as an einsum
    print(torch.einsum('ij, jk -> ik', A, B))
  • model.train() enables stochastic/training-specific behavior like dropout and batchnorm.

  • Use .to(device) on the models and inputs, not in the forward method. The standard pattern is

    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    model = MyModel().to(device)

    for imgs, labels in dataloader:
    imgs, labels = imgs.to(device), labels.to(device)
    out = model(imgs)
  • nn.TransformerDecoder implements the crossattention step outlined in the 2017 paper. TransformerEncoder with casual mask is what we want for GPT style decoder.
