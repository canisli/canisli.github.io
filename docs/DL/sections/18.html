18. Applying Deep Learning to Your Problems

18.1. Data

  • Check input and output dimensions
  • Preprocess: augmentatino, resizing, feature engineering, tokenization
  • Post-process: NMS, bipartite matching, detokenization

  • Estimate problem scale via total data dimensionality and total information content.

    • CV uses 100-100 epochs, LLMs often use 1

18.2. Model

  • Prototyping Models:

    • Brute force match shapes, e.g. turn audio into 2D spectrogram and treat it like an image and apply masked autoencoder.

    • Start small

      • MLP: 3–5 layers, 32–256 units, ReLU.
      • ConvNet: LeNet-like, or ResNet-10/18.
      • Transformer: 3–5 blocks.
      • Reduce channels (like 32).

  • Metrics for model scale

    • number of parameters
    • FLOPs for a forward or forward+backward pass
    • For transformers at fixed sequence length, params and FLOPs are roughly proportional

  • Always do feasibility survey when you start a project

    • What is “standard”. What is “SOTA”
    • What scales can you afford
    • What’s your expeirmental cycle

18.3. Optimizer

  • LR is most important. LR and batch size are not independent. Search by octave, by multiplying and dividing by factor 2. Fix the batch size during prototyping as it affects the loss geometry. Then when you decide to change batch size for scaling, scale learning rate linearly or sqrtly.

  • LR schedule

    • cosine decay for recognition problems
    • const for generation problems

  • LR warmup

  • weight decay has two notions

    • In a classical sense: L2 regularization on the weights
    • Modern: decay the weights e.g. AdamW (this implements classical weight decay)

  • Adam momentums

    • beta1 = 0.9
    • beta2 = 0.999 by default, but strongly recommended to use 0.95

  • Weight initialization strongly affects optimization

    • For ReLU use Kaiming initialization
    • For tanh/sigmoid/linaer layers use Xavier/Glorot
    • For some transformer models use Gaussian with fixed std
    • For some special layers like bias, initialize them to zero.
  • Fit and overfit a tiny problem

18.4. Monitoring

  • Monitor metrics, norms of gradients, weights, activations, variance of metrics, time/speed
  • Visualize outputs and intermediates
