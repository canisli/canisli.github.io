Midterm Prep

  • Topics covered

    • Environment diagrams (indicate type!)
    • Functions

    • Recursion and path finding

    • Labs: image processing, bacon number, snekoban, autocomplete, mice-sleeper game

  • Turn lists and sets into tuples and frozensets if you want to hash them.

    • list(…), set(…), dict(…) does a shallow copy.
  • Recall that when we make a new frame for a function call, the new frame’s parent pointer goes to the function’s enclosing frame (i.e., the frame where that function object was defined).

  • Gotchas:

    • Use set() to denote empty set instead of {}.
    • Tuple comprehensions do not exist. (…) is a generator
    • Recursion limit 1000
    • Use (a,) to denote a singleton tuple
    • After a for loop the variable is still in stack.

  • Common Errors

    • NameError: name 'y' is not defined.
    • IndexError: list index out of range.
    • RecursionError: maximum recursion depth exceeded

    • ValueError: right type but invalid value

      • ValueError: invalid literal for int() with base 10: 'abc'
      • Exception: ZeroDivisionError

    • TypeError: wrong type for an operation or wrong number of arguments

      • TypeError: f() takes 0 positional arguments but 1 was given
      • TypeError: unsupported operand type(s) for +: 'int' and 'str'
